## 🔒 Features to LOCK Behind Paid Tiers (Enforce in-app)

Gate at the **feature flag layer** in code (`settings.enabledFeatures`):

### Behind Pro+
- `enabledFeatures.purchaseOrders` → Hide Purchasing nav item
- `enabledFeatures.deliveries` → Hide Deliveries nav item
- `enabledFeatures.samples` → Hide Samples nav item
- `enabledFeatures.tvSlideshow` → Disable TV mode
- `enabledFeatures.customerRouting` → Customer profile hides the "Custom workflow" checkbox
- `enabledFeatures.kanbanBoard` → Hide Board nav item
- `enabledFeatures.shiftAlarms` → Hide Schedule→Alarms section in Settings
- `enabledFeatures.workerBadges` → Hide badge print button in Team

### Behind Shop OS only
- `enabledFeatures.quality` → Hide Quality nav
- `enabledFeatures.reports` → Hide Reports nav
- `enabledFeatures.financials` → Hide Financial settings tab
- `enabledFeatures.aiScanning` → Disable AI PO scanner button
- `enabledFeatures.apiAccess` → Hide API section in Settings
- `enabledFeatures.sso` → Hide SSO config
- `enabledFeatures.customDomain` → Hide custom-domain portal config

### For every locked feature
- Show a **friendly upgrade nudge** instead of hiding entirely on hover. Example card:
  > *"🔒 Purchase Orders is part of Pro. [Upgrade]"*
- When clicked, takes user to `/billing/upgrade?feature=purchase-orders`

## 🛠 Technical Notes for Landing Page Build

### Framework suggestion
- **Astro** or **Next.js** — static HTML for speed, SEO-friendly
- Tailwind for styling (matches app DNA)
- Hosted on **Netlify** (we already use it — `netlify.toml` is in repo)

### Domain
- Primary: `shopos.app` or `shopos.io` (check availability)
- App stays at: `app.shopos.app`
- Customer portals: `{slug}.shopos.app` or `shopos.app/c/{slug}`

### Must-have pages
- `/` — Landing (spec above)
- `/pricing` — Deep-dive of the 3 tiers
- `/features/jobs` — Deep-dive
- `/features/customer-portal` — Deep-dive with live demo
- `/features/quality` — Deep-dive
- `/demo` — Interactive sandbox (click-through of real app with seed data)
- `/docs` — Help docs
- `/changelog` — What shipped this week (marketing flywheel)
- `/privacy` · `/terms` · `/contact`

### Analytics
- Plausible or PostHog (not GA — privacy-first matches the "your data is yours" pitch)
- Track: hero CTA clicks, pricing hover, trial signup, demo watch completion

### Signup flow
1. Landing → "Start free trial"
2. Email + password (or Google SSO)
3. Shop profile wizard (industry, size, certs) — 5 questions
4. Firebase Auth creates user + seeds `settings.shopProfile`
5. Redirect to `app.shopos.app/overview`
6. 14-day trial, day 12 reminder, day 14 soft paywall
7. Billing via Stripe Checkout → writes `settings.subscription` to Firestore

### Demo-mode data
Ship a "Play with a sample shop" option:
- Uses a read-only seed dataset (dummy customers, 50 jobs, 5 workers)
- No login required, 10-minute timeout
- Clicking "Save" anywhere pops a "Create your free account to keep this" modal
